Elements

As mentioned on the previous section, the combination of an opening and a closing HTML tag of the same type forms an HTML element.
It's easy to get both terminologies confused, so remember that an HTML tag is part of the syntax that creates an element, which is an entity with potential content and a display value (as we will see later in the chapter).

Syntax

Although the above fits in a single line, elements can be much greater, spreading accorss multiple lines of code. Some called container or parent elements can in fact contain one or multiple elements within their content, called child elements, for which the term nesting is often employed.

So with that in mind, let's now back-track a bit, to our first HTML syntax sample from the previous section, and try to make sense of its tags, the elements they create, and several other very useful ones.

Elements To Remember

<!doctype html> the declaration tag must be the very first tag in an HTML document. Although technically not an HTML tag (which explains the ! before the name), it informs the web-browser about the HTML version used in the document and requires no closing tag.
Note that the one used here is the generic version for HTML5, and although html is typed in lower-case, doctype can be typed in either lower-case or capitals.
<html></html> the html tags forming the root element define the length of the HTML document, indicating to the browser where the HTML code starts and ends.
<head></head> the head tags forming the head element provide information about the HTML document not to be displayed, such as its title, set of characters used (alphabet), links...
<title></title> the title tags forming the title element are required in all HTML documents, and must always be placed within the head element since their main purpose is to name the document.
<body></body> the body tags forming the body element define the area where all displayable elements are contained (articles, paragraphs, images...), and must always be placed immediately after closing the head element.
<h1></h1> there are six different sizes of h tags forming the heading elements, from <h1> to <h6> (bigger to smaller), to differentiate important headings from others.
<p></p> the p tags forming the paragraph element are used to create text paragraphs.
<div></div> the div tags forming the div element are mainly used to nest other elements as a container.
<a></a> the a tags forming the hyperlink element, connect the page to another chosen document where the user is then redirected to, by clicking on the link.
<ol></ol> the ol tags forming the ordered-list element are used to create an ordered list, be it alphabetically or numerically, and require the use of <li> tags to define each of its items.
<ul></ul> on the contrary of ol, the ul tags which form the unordered-list element create (you guessed it) an unordered bulleted list, and here too the use of <li> tags is required to define all items.
<li></li> the li tags forming the list-item element define as mentioned a list item.
<table></table> the table tags forming the table element create an HTML table that uses the <tr>, <th>, and <td> tags to define (in the same order) its rows, headers, and cells content.
<tr></tr> the tr tags which forms the table-row element define a row in an HTML table, and may use the <th> and <td> tags to define both its header and cells content.
<th></th> the th tags forming the table-header element define a cell or row's header in an HTML table.
<td></td> the td tags form the table-data element, defining each cell's content in an HTML table.
<header></header> the header tags forming the header element (not to be confused with the head element) are used to create a container such as a navigation bar, assumingly at the top of the page or the parent element.
<footer></footer> on the contrary, the footer tags forming the footer element create a container normally used for the lower part of the page or the parent element.
<!-- comment --> the comment tags allows for comments to be included within the HTML code, without being processed by the web browser (they are simply ignored by it).
Very handy as the HTML document grows in size, they can also be used to explain parts of your code to other coders when working as a team.
