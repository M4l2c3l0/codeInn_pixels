Void Elements

Since there are no rules without exceptions, I now introduce you to void elements.
Void or empty elements have no content and no closing tag (...come again?). All of their information is in fact defined via attributes, which are here compulsory.
Rest assured however as there are not that many out there, and to make sure it is all as clear as possible, we will go through some of the most commonly used ones.

Syntax

Similar but not quite, right?
The noticeable differences are the absence of content and closing tag.
Void elements do not have any content, consequently causing their syntax evolution to remove their closing tag.
Thinking of it, their structure is actually rather simple, for it consists of a single tag with at least one attribute in it. As you will in fact realise, the main thing to remember here is what attributes go with what void elements.

Common Void Elements

<meta> provides metadata (information about data) to the browser about the HTML document. Not to be displayed, any meta tag belongs within the head section of the HTML document:
<meta charset="utf-8"> (specifies the character encoding, chosen in accordance to the characters used for the webpage's alphabet)
<link> defines a link between the HTML document and an external resource, commonly used for stylesheets (such as CSS documents):
<link rel="stylesheet" href="url"> (the rel attribute specifies the relationship between the external source to the HTML document, and the href provides the url)
<img> links an image to the HTML page which is then displayed by the browser. It requires the src attribute to provide its directory and the alt one to give it a description:
<img src="directory" alt="description">
<br> the line-break tag produces a line-break in the text:
<p>First line<br>Second line</p>
(output:)
First line
Second line
<input> creates an area for the user to enter data (information):
<input type="text" name="joke"> (creates a text area for the user to enter a joke)
