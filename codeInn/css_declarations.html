CSS/ Declarations

Declarations

Declarations define all changes to be applied to the HTML element(s), by defining which of their propertie(s) to modify and assigning a new value to it (which as you can imagine gives you tremendous control over an element's style, but mind all that new power of yours!).
In code, less is more, and often better too.
As you apply style declarations to your elements, you often realise that many are exactly the same (with repetitive property-values), so it is also good practise to combine those into a shared declaration block targetting the concerned elements:
h1 {color: black;}
p {color: black;}
(become:)

h1, p {
  color: black;
}

Style Declaration

A CSS declaration always starts with the property name in plain US english (not UK!), written in lower-case and, if formed of multiple words, a dash is used in-between.
A colon separates the name from its new value, and since (as for HTML) whitespace are here ignored by the browser, they are a purely personal preference.
Finally, the declaration is completed by a semicolon after the value, which technically also separates it from any other declaration tha may come after.
Note that certain properties require multiple values, so only after all are entered should the closing semicolon be placed.

Good To Know

width defines the element's width, in either pixels or percentage related to the parent's one:
width:150px; (defines width to 150px)
width:50%; (defines width to 50% from its parent's)
height defines the element's height, and it too can be in both pixels or percentage of the parent's height:
height:80px; (defines height to 80px)
height:30%; (defines height to 30% from its parent's)
border also called shorthand, this property allows you to define the thickness, style and colour of an element's border all around in a single declaration. Side can be singled out by adding top, right, bottom, or left to the property name:
border: 1px dotted blue; (1px thick continuous blue border)
border-top: 2px dotted red; (2px thick dotted red top border)
padding creates transparent space between the element's content and its border. Here too a side can be singled out by adding any of the four sides to it, however padding's value can also be specific enough to assign a different value to each side without declaring them individually, following a clockwise order starting at the top:
padding-right: 5px; (5px on right)
padding: 5px; (5px on all four sides)
5px 10px; (5px top and bottom, 10px right and left)
5px 10px 0; (5px top, 10px right and left, none bottom)
5px 5px 0 0; (5px top, 5px right, none bottom, none left)
margin creates transparent gap between the element's border and the surrounding ones. The same values than padding apply here too, however it can also be used to centre the element in relation to its parent's width:
margin-bottom: 5px; (5px on bottom)
margin: 5px; (5px of gap all around)
5px 10px; (5px top and bottom, 10px right and left)
0 auto; (none top and bottom, as much as possible and equally balanced on left and right, placing the item at the centre)
color although it can lead to confusion, this property defines the text colour rather than the element's. It can be defined by one of the colour names ragistered (about 150 possibilities) or by numeric values combination such as Hexadecimal, rgb/rgba, or hsl/hsla (in both cases, the a stands for alpha, defining the transparency):
color: gray;
#808080; (hexadecimal value)
rgb(128,128,128); (rgb value)
hsla(60,100%,98%,0.81); (hsla value with transparency)
background being another shorthand, it allows you to define an element's background properties in a single declaration, such as is colour, background image, its positioning, etc.
All of course can be declared separately:
background-color: #ccc; (defines background colour) background-image: url("directory); (provides background image) background: #ccc url("directory) center no-repeat; (defines background colour and image, then centered and not repeated)
font here we have another shorthand which therefore will require several values, defining both the size in px, % or em, separated by at least a whitespace from the font family value, for which you may have multiple possibilities, (left being first) all separated by a comma from each other.
Each part can however be defined in separate declarations by either adding size or family to the property:
font: 1.2em Arial, sans-serif; (font size 1.2 times the current one, in Arial, and if not available display any font from sans-serif family)
font-family: PT Mono, monospace; (font in PT Mono, and if not available display any font from monospace family)
font-size: 20px; (font size of 20px)
