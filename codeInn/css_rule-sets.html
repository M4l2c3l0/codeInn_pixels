CSS/ Rule Sets

Rule Sets

The main purpose of CSS is basically to target specific HTML elements and apply some make-up to them in two steps: by selecting them, and then modify the disired properties.
As we are about to see in more detail, the combination forms a CSS rule set, and there can be as many as you wish in a CSS stylesheet document.
It is for this reason that it is good practise to keep your CSS code organized and tidy from the start, by leaving for example separation lines between rule sets, writing them over multiple lines instead of a single one, and break each new declaration to the following line.
To get a better visual of it, let's take a look at a CSS rule set, its different parts, and understand how to present it following the best practises above.

Syntax

Each rule set starts by targetting one or multiple elements via the selectors, and is then followed by the declaration block, containing all the styling declarations wrapped in-between two curly braces {}.
Although the above is on a single line, it is good practise to write it on multiple lines, increasing its visibility and facilitating any error handling that may be required:
h1, p {
  font-size: 14px;
  color: black;
}
(both are correct, but the multi-line version is much cleaner and easier on the eye)

So let's now dive into its syntax in more details starting, from left to right, by the selectors.
As we have already seen, we can select elements by their type, and any style declaration defined will be applied to all elements of that/those type(s). But as you are now about to learn, there are several other selectors that can be used and offering you better control over your selection.

Pick Me .Me #Me

type the usage of the element's type as selectors is simple and broad, applying any changes to all elements of the chosen type(s). A commacan be placed between multiple and different types, allowing you to select all elements of those types:
h1, h2, p
(targets all h1, h2 and p elements)
.class when targetting multiple items (not necessarely of the same type, and not necessarely all of a particular type), a class attribute may be given to the desired ones, and its name (value given to the class attribute) used as a selector, by simply placing a dot before its name:
.className
(targets all elements with that class name)
#id an item can be singled out by givin an id attribute to it and use that with a hashtag # before its name.
Remember that an id must remain unique within an HTML document:
#idName
(singles-out element with unique id)
nesting flow by removing the comma for example between several elements (say div and p), we can take advantage of the elements nesting levels, which from left to right reads as parent to child, baring in mind that the one on the far right is the target:
div p
(targets all paragraphs inside all div containers)
