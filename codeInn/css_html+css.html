HTML + CSS

Although it seems that two decades ago HTML was rather independent (it was in fact common for websites throughout the 90s to be designed purely by using HTML alone), today's web design standards certainly strengthened its dependency on CSS to deal with that matter, so much so that HTML itself evolved accordingly, adapting and even removing several of its styling tags.

Pimp My HTML

<!doctype html>
<html>
  <head>
    <title>Page Title</title>
    <style>
      h1, p {
        colour: green;
        font-size: 14px;
      }
    </style>
   </head>
  <body>
    <h1>Big Heading</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

Here we have an example of what is called internal CSS, which consists of placing the CSS code inside of the head section in the HTML document, contained by a pair of two <style> tags.
Just like the <html> tags used before, HTML uses the <style> ones to inform the browser that anything in between is written in CSS.
So is this the only method to provide some CSS into an HTML document?
There are as you are about to see different ways to do so, all with their pros and cons, and although one may be more efficient than others depending on each particular case, the choice is ultimately defined by our personal style and preferences.

Implement CSS

Inline CSS can be placed at the core of an element to target it alone, by placing its declarations within the element's opening tag as part of the style attribute. Just like for any attribute's value(s), these declarations will be wrapped by two single or double quotes, and separated from each other by at least one whitespace:
<h1 style="colour:gray; font-size:18px;">
(defines colour and size of heading)
Internal the CSS code can also be placed within the head section of the HTML document, and can therefore style any element within the documents:
(method used for the example above)
.css linked file a separated CSS stylesheet can be created and saved with the .css extention. To connect it to an HTML document, a <link> tag must be placed in the head section. This tag must also contain the rel attribute specifying the document's relation (stylesheet for CSS), and the href one to indicate its directory.
This is great for bigger websites with multiple pages, as all can linked and therefore commonly affected by the same style declarations defined in a single CSS stylesheet.
Note & remember: all of a website's documents must be inside the same folder, referred to as the root folder. If not, the HTML document will simply not be able to follow the provided directory:
<link rel="stylesheet" href="style.css">
(links to a target of type stylesheet, and provides its directory)
