CSS/ Cascading StyleSheet

Back To The Start

Designed to separate a document content from its presentation, CSS is a stylesheet language used for the presentation of a document written in a markup language, defining how to display HTML elements.

Proposed by HÃ¥kon Wium Lie in 1994, at the time working with Tim Berners-Lee (rings a bell?) at CERN, it was officially released two years later after the contribution of Bert Bros, who would later be regarded as its co-creator.
Its latest standard CSS3 is described as being completely backwards-compatible to other CSS versions, as it aims to extend the previous CSS2, including some great styling novelties such as animations or new layouts (amongst many others).

First Impression

h1, p {
  colour: green;
  font-size: 14px;
}

Although the CSS syntax may appear quiet simplistic at first, it can quickly become a headache to deal with as your code grows (believe you me!).
No more markup tags here, however notice several similarities to our previous HTML chapter, such as some HTML tag names (here, h1 and p), and two lines of code that seem very similar to the attributes used in HTML tags, wrapped in-between an opening and closing curly braces {}.

But before going any further into the details of its syntax, let's backtrack here for a moment and take a look at the bigger picture; how to implement CSS code into an HTML document.
